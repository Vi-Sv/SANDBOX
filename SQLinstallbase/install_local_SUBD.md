# Ультимативная инструкция: Автономная (Portable) аналитическая лаборатория

Данная инструкция позволяет собрать полностью независимый комплект из СУБД **PostgreSQL 16** и графического интерфейса **DBeaver**. Комплект работает на любом компьютере с Windows, не требует прав администратора, не засоряет систему и запускается прямо с флешки или внешнего диска.

---

## Шаг 1: Скачивание чистых компонентов (без инсталляторов)

Скачайте два ZIP-архива. **Не запускайте установку**, нужны именно файлы внутри архивов.

1. **Ядро СУБД (PostgreSQL 16.14)**:
   * https://www.enterprisedb.com/download-postgresql-binaries
   * https://sbp.enterprisedb.com/getfile.jsp?fileid=1260308
     
2. **Графический интерфейс (DBeaver CE 26.1.2)**:
   * [Скачать архив dbeaver-ce-26.1.2-windows-x86_64.zip](https://github.com/dbeaver/dbeaver/releases) *(Официальный репозиторий GitHub)*
   * https://github.com/dbeaver/dbeaver/releases/download/26.1.2/dbeaver-ce-26.1.2-windows-x86_64.zip

---

## Шаг 2: Создание структуры папок

Перенесите будущую сборку в корень любого диска (например, `C:\` или `D:\`).
* **Критически важно:** в пути к папке **не должно быть русских букв и пробелов**, иначе база данных выдаст ошибку кодировки при инициализации.

Создайте следующую структуру директорий:
* 📁 **`C:\Portable_Data_Suite\`** (Корневая папка лаборатории)
  * 📁 **`data`** — *Создайте эту папку вручную. Оставьте её абсолютно пустой.* Здесь будут физически храниться ваши таблицы и базы.
  * 📁 **`dbeaver`** — *Распакуйте сюда содержимое архива DBeaver.* (Внутри должен лежать файл `dbeaver.exe`).
  * 📁 **`pgsql`** — *Распакуйте сюда содержимое архива PostgreSQL.* (Внутри должна появиться папка `bin` и другие файлы СУБД).

---

## Шаг 3: Создание пускового файла автоматизации

В корневой папке `C:\Portable_Data_Suite\` создайте обычный текстовый файл и переименуйте его в **`start_database.bat`** (убедитесь, что расширение изменилось с `.txt` на `.bat`).

Нажмите по нему правой кнопкой мыши -> **Изменить** (или открыть в Блокноте) и вставьте этот точный код:

```batch
@echo off
chcp 65001 > nul
SET "PGPATH=%~dp0pgsql\bin"
SET "PGDATA=%~dp0data"
SET "PGPORT=5433"

:: Проверяем первый запуск. Если папка пуста — инициализируем кластер БД.
if not exist "%PGDATA%\PG_VERSION" (
    echo [СИСТЕМА] Обнаружен первый запуск. Инициализация новой базы...
    "%PGPATH%\initdb.exe" -U postgres -A trust -E UTF8
    if errorlevel 1 (
        echo [ОШИБКА] Не удалось инициализировать базу данных.
        pause
        exit /b
    )
    echo [СИСТЕМА] База данных успешно создана в папке \data\
    echo.
)

echo [СИСТЕМА] Запуск сервера PostgreSQL на выделенном порту %PGPORT%...
"%PGPATH%\pg_ctl.exe" -D "%PGDATA%" -o "-p %PGPORT%" -l "%~dp0server.log" start

if errorlevel 1 (
    echo.
    echo [ОШИБКА] Сервер не смог запуститься. 
    echo Проверьте файл server.log в папке со скриптом.
    pause
    exit /b
)

echo.
echo #######################################################
echo  ПОРТАТИВНАЯ БАЗА ДАННЫХ УСПЕШНО ЗАПУЩЕНА!
echo.
echo  1. Откройте DBeaver из соседней папки.
echo  2. Подключитесь к хосту: localhost
echo  3. Укажите ПОРТ (Port): %PGPORT%
echo  4. Пользователь: postgres
echo  5. Пароль: оставить ПУСТЫМ
echo.
echo  НЕ ЗАКРЫВАЙТЕ это окно во время работы в DBeaver.
echo #######################################################
echo.
echo Чтобы БЕЗОПАСНО остановить сервер, нажмите любую клавишу...
pause > nul

echo.
echo [СИСТЕМА] Остановка сервера PostgreSQL...
"%PGPATH%\pg_ctl.exe" -D "%PGDATA%" stop
echo [СИСТЕМА] Сервер остановлен. Теперь окно можно закрыть.
timeout /t 3
```
Сохраните файл (**Ctrl + S**) и закройте Блокнот.

---

## Шаг 4: Первый запуск и развертывание СУБД

1. Дважды кликните по файлу **`start_database.bat`**.
2. В самый первый раз скрипт автоматически создаст конфигурационные файлы базы данных внутри папки `data`.
3. На экране появится надпись: `сервер запущен`. **Не закрывайте это черное окно консоли**, пока работаете с данными. 
* *Примечание:* Сервер СУБД намеренно запущен на порту **5433**, чтобы исключить конфликты, если на компьютере уже установлена другая стандартная копия Postgres.

---

## Шаг 5: Подключение графического интерфейса (DBeaver)

1. Зайдите в папку `C:\Portable_Data_Suite\dbeaver\` и запустите **`dbeaver.exe`**.
2. В левом верхнем углу нажмите кнопку **Новое соединение** (иконка розетки с зеленым плюсом).
3. Выберите **PostgreSQL** -> нажмите **Далее**.
4. В окне настроек на вкладке *Главное (Main)* укажите следующие параметры:
   * **Хост (Host)**: `localhost`
   * **Порт (Port)**: измените стандартный 5432 на **`5433`**
   * **База данных (Database)**: `postgres`
   * **Пользователь (Username)**: `postgres`
   * **Пароль (Password)**: полностью сотрите, поле должно быть **абсолютно пустым**.
5. Нажмите кнопку **Тест соединения** в левом нижнем углу. При первом подключении DBeaver предложит подгрузить драйвер — нажмите **Скачать (Download)**. 
6. После появления информационного окна «Соединено» нажмите *ОК* и затем *Готово (Finish)*.

---

##  Правило безопасного завершения работы (Важно!)

Когда вы закончили работу в DBeaver и хотите закрыть базу или перенести папку:
1. Полностью **закройте программу DBeaver** (чтобы разорвать активные сессии).
2. Перейдите в открытое черное окно консоли `start_database.bat`.
3. Нажмите **любую клавишу** (например, Пробел). Скрипт корректно выгрузит сервер из памяти, запишет кэш и сохранит все таблицы на диск. Окно закроется само.

Теперь папка `Portable_Data_Suite` находится в безопасном «спящем» состоянии. Её можно архивировать, отправлять в облако или копировать на флешку. Локальная инфраструктура для обучения анализу данных полностью готова.
