## ШАГ 1: Развертывание локальной инфраструктуры и первая ручная микропрактика

Срок этапа: 3–5 дней (в режиме плавного капельного входа без перегрузки).
------------------------------
## 1. Инфраструктурная сборка (Что нужно установить на ПК)
Тебе запрещено использовать онлайн-тренажеры или облачные песочницы. Аналитик должен уметь разворачивать рабочую среду сам. Мы ставим промышленный стандарт:

   1. СУБД PostgreSQL (Версия 15 или 16):
   * Зачем: Твое локальное хранилище данных, которое будет крутиться на твоем процессоре и RAM.
      * Действие: Скачиваешь официальный бесплатный дистрибутив (для Windows/Mac). При установке СУБД попросит задать пароль для суперпользователя postgres. Запиши его на бумагу. Это твой главный ключ. Порт оставляешь дефолтный — 5432.
   2. DBeaver Community Edition (CE):
   * Зачем: Твое визуальное рабочее окно (IDE). Программа, через которую ты будешь писать SQL-запросы, отправлять их в PostgreSQL и видеть таблицы результатов.
      * Действие: Скачиваешь и устанавливаешь. Запускаешь, выбираешь «Создать новое соединение», выбираешь PostgreSQL. Вводишь имя пользователя (postgres) и тот самый пароль, что задал при установке СУБД. Нажимаешь «Тест соединения». Если загорелся зеленый маркер — мост построен.
   
------------------------------
## 2. Практическая цель: Создание первой «песочницы» руками
Ты должен понять, как база устроена физически. Внутри DBeaver ты создаешь пустой SQL-скрипт и выполняешь три последовательные операции. Каждое слово ты вбиваешь руками, копирование из ИИ запрещено.

* Операция №1: Создание независимой базы данных.

CREATE DATABASE sandbox_db;

(Ты создал изолированный бак для экспериментов).
* Операция №2: Создание первой простейшей таблицы.

CREATE TABLE test_logs (
    record_id SERIAL PRIMARY KEY,
    event_name VARCHAR(100),
    event_value NUMERIC(10,2),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

(Ты разметил структуру: уникальный автосчетчик строк, текстовое поле, точное число с 2 знаками после запятой и временная метка, которая ставится сама).
* Операция №3: Ручной запуск данных в бак.

Понимание команды CREATE TABLE и структуры метаданных — это не прихоть админа, а базовая необходимость для аналитика данных по трем практическим причинам:

   3. Создание временных таблиц в расчетах. Для тяжелых отчетов аналитики часто создают промежуточные таблицы на лету, чтобы сохранить туда кусок данных, посчитать его и удалить.

Ты делаешь это один раз, чтобы своими глазами увидеть, как данные распределяются по ячейкам памяти на твоем ПК. Дальше мы будем заниматься только извлечением данных (SELECT).

   1. Ты прошел 3 сквозных боевых симуляции «запрос -> ТЗ -> код -> дашборд -> защита» без паники и зависаний.
   2. Ты четко понимаешь ИТ-процессы, ориентируешься в терминах (спринт, бэклог, дедлайн, дата-дискавери) не потому, что зазубрил их, а потому, что руками потрогал их в симуляции.
   3. Твое резюме открыто на рынке, и ты бьешь по вакансиям подготовленной, хладнокровной и технически сильной боевой единицей.
